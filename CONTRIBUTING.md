# Contributing to the Poisonous Mushroom Predictor Project

This outlines how to propose a change to the Poisonous Mushroom Predictor project. Contributions are welcome since we realize the importance of change and improvement and aim to provide the best version of this project with the most useful results. If you have suggestions, feel free to leave a message or leave a comment for any of the core group members, we know how important it is that get feedback from others.

## Ways to contribute

- Make suggestions for improvement
- Flag typographical errors
- Report bugs, or unexpected behaviour
- Suggest desired features
- Help review and test the code
- Leave a message or make a new issue

## Prerequistites & Code of Conduct

Please ensure that you review and follow the code of conduct stated in the [Contributor Code of
Conduct](CODE_OF_CONDUCT.md) file in the project repository root before contributing to this project.
You must follow the instructions and guidelines as a data analyst and demonstrate professionalism and appropriate conduct as defined therein, when contributing to this project. Thank you in advance for your cooperation!

## Process for submitting feedback

- Reach out to a core group member about your object of interest - bug fix, new feature, any other suggestions.
- If your proposed change is substantial, please ensure a core group member agrees before creating a pull request. 
- Create a separate branch to work on the code and submit a detailed and clear pull request for review. 
- Please use the PEP8 [style guide](https://peps.python.org/pep-0008/) and include tests and documentation where applicable. 
- All pull requests will be reviewed through a fair and thorough process and the contributor will be credited appropriately on a case-by-case basis.
